# Security Policy

## Supported versions

Security fixes are applied to the latest released version of `pi-openspec-status` on npm. Older releases may not receive fixes; please upgrade to the newest version before reporting an issue when possible.

## Reporting a vulnerability

Please **do not open a public issue** for a suspected vulnerability. Report it privately through GitHub's [private vulnerability reporting flow](https://github.com/richards-ensono/pi-openspec-status/security/advisories/new). Repository administrators must keep private vulnerability reporting enabled as described in [maintainer security settings](.github/MAINTAINERS.md).

Include a clear description of the issue, affected versions, reproduction steps or a proof of concept, impact assessment, and any suggested mitigations. Do not include credentials, tokens, or other sensitive data.

## Response expectations

Maintainers aim to acknowledge a report within five business days and will provide a status update after assessing the report. Resolution timing depends on severity, reproducibility, and release coordination requirements.

## Responsible disclosure

Keep vulnerability details private until maintainers confirm that a fix or mitigation is available and agree on a disclosure timeline. Please avoid accessing, modifying, or deleting data that does not belong to you, and test only against systems you are authorized to assess.
